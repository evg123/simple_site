Port Mortem

I'm happy with what I accomplished on Breakout during the allocated time, but there are a number of things that could be improved.

If I had an extra eight weeks to work on this project and get it ready for release the first order of business would be adding proper resolution support. Currently the game is hardcoded to 720p and doesn’t support resizing the window. In order for this to work I would need to decouple the placement of bricks from the resolution of the window so that a level config file would look the same with any window size and shape.

Another improvement to get this game release-ready is improving the build system and adding support for other OSes. The current build script is very fragile and I haven’t tested it at all on platforms other than OSX. I would look into other build system technologies, but just a proper makefile would be a big improvement.

After that I would focus on improving the level config file format, which is currently pretty bare-bones. It allows the placement of bricks and balls where ever you want, but lacks the ability to specify different sizes or colors for individual blocks. It is also not a very user-friendly format. The addition of a simple GUI system for generating level config files would make level creation much easier. It wouldn't have to be too complicated to give users a lot of freedom. A way to add new blocks and drag them around, and the ability to select individual blocks and set their properties (color, size, movement direction) would be enough.
